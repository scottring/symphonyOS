## Demo Tips

### Pacing
- **0:00-1:00** ‚Äî Set up the problem and promise (talk fast, build energy)
- **1:00-4:00** ‚Äî Show features (slow down, let UI breathe, less talking)
- **4:00-5:00** ‚Äî Vision and close (confident, clear, memorable)

### Visual Flow
- Start desktop (shows power and planning)
- Switch to mobile (shows execution simplicity)
- End desktop (shows the full system)

### Interaction Style
- **Don't narrate every click** ‚Äî Let the UI speak for itself
- **Do explain the why** ‚Äî "Notice how the project context appears automatically"
- **Do show real scenarios** ‚Äî Use realistic task names and context

### Common Mistakes to Avoid
1. Spending too long on any one feature
2. Talking while clicking (pause after click, let them see, then explain)
3. Apologizing for incomplete features
4. Getting lost in technical details
5. Forgetting to show mobile view

## Post-Demo Questions to Expect

**Q: "Can I share tasks with my spouse?"**
A: "Yes! Family domain tasks are shared with everyone in your household. Each person sees the same tasks and can mark them complete. Work and personal domains stay private."

**Q: "Does this replace my calendar?"**
A: "No, Google Calendar is your source of truth. Symphony reads from your calendar and shows events alongside tasks. We don't replace your calendar ‚Äî we enhance it with tasks and context."

**Q: "What about recurring tasks?"**
A: "Routines are coming in V1.6. Right now you can manually reschedule recurring items, but automated recurrence is on the roadmap."

**Q: "Is this available now?"**
A: "We're in active development. V1.5 is live with the core features you just saw. We're adding more capabilities every sprint."

**Q: "How much does it cost?"**
A: "We're currently in private beta ‚Äî free while we refine the experience with early users."
